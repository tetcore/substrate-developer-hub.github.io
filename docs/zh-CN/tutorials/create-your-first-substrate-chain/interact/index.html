启动节点

# 在开发模式运行一个临时节点
./target/release/node-template --dev --tmp

启动前端

要与您的本地节点交互，我们将使用 Substrate 开发者 Front-end Template，它集合了常规情况下需要的各种前端交互组件。

如果您已经安装了 Front-end Template，请在 Front-end Template 所在的根目录中执行以下命令：

# 确保在 Front-end Template 根目录中运行此命令
yarn start

交互

当 Front-end Template 成功运行后，浏览器会载入 http://localhost:8000/ 这地址，请花点时间来探索它的组件吧。 在顶部， 您会看到关于您连接着的区块链的有用信息，以及 一个帐户选择器，来选择您在链上执行操作时使用的帐户。

这里也有一个表格，列出了您可调动的 常用的测试账户。 一些像 Alice、Bob 等的账户已经都有资金了！

在账户列表下面，您可以使用转账组件从一个账户转账到另一个账户。 注意 Front-End Template 使用精度的信息提示； 您应该传输至少 10000000000 个代币来方便观察您正在做的更改。

请注意，账户列表里的数据是动态获取的，在处理转账后账户余额会立即更新。

Runtime 元数据

Front-End Template 显示着许多有用的功能，当您在连着本地开发节点时，您应该好好探索所有这些功能。 一个很好的开始方式是点击模板页面顶部的 “显示元数据” 按钮查看 Runtime 的元数据。

模块交互组件 & 事件

您可以使用 runtime 元数据来发现 runtime 的功能。 Front-End Template 提供了一个模块交互组件，它提供了几种与 Substrate 进程进行交互的方式。

Extrinsics 是 runtime 里可调用的函数。 如果你熟悉区块链概念，你可把它们看作是链上交易。 模塊交互組件允许您提交 未签名 或 已签名 的交易，并且提供了一个按钮，可轻松提交交易 使用 Sudo 模块里的 sudo 函数。 在第三個教程 添加一个Pallet 教程裡，您将了解更多关于使用 “Sudo” 按钮来调用需要特权的交易。

您可以选择查询交互来读取 Runtime 存储中的值 。 RPC 和常备选项为 runtime 的提供了额外的交互机制。

正如许多区块链一样, 基于 Substrate 的链使用 事件（Event） 来报告异步操作的结果. 如果您已经使用了 Front-End Template 来执行上述的余额转账， 您可以在 “Pallet 交互組件” 旁边的 “事件组件” 中看到一个转账事件。
