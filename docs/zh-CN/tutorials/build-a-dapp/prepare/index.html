关于存证

我们接下来将构建的 dapp 是一个存证服务 (PoE)。 来自 维基百科:

存证是一种在线服务，通过比特币区块链中的时间戳交易验证一个计算机文件是否在某个特定时间段里存在。

一般用户都不需要上传整个文件到区块链来 “证明其存在”，而是通过提交一个 文件哈希值，或称之为文件摘要或校验。 这些哈希是非常有用的，因为巨型文件可以用一个小的哈希值来唯一表示，这个值对于存储在区块链上是非常有效率的。 任何拥有原始文件的用户可以简单地通过重新计算文件的哈希值并与区块链上存储的值相比较来证明该文件与区块链上的是一致的。

另外，区块链系统还提供了一个强大的用户认证系统。 所以当文件摘要存储进区块链时，我们也同时记录是哪个用户上传了该摘要。 这使得该用户在后续能够证明他是该文件的拥有人。

接口与设计

我们的 PoE 接口将开放两个可调用的功能：

  • create_claim - 允许用户通过上传文件摘要来声称文件存在。
  • revoke_claim - 允许当前的声明拥有人撤销他们的拥有权。

为了实现这一点，我们只需要储存已经声称的证明以及是谁提出这些的证明即可。
