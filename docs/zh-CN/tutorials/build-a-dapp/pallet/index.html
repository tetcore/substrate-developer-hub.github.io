构建一个自定义模块

Substrate 开发者平台的 Node Template 将作为此教程的起点，有着基于模块的 Runtime。 FRAME 是一系列的代码库，它允许 通过模块 (叫 "pallet") 来构建一个 Substrate Runtime。 你可以将这些模块视为独立的逻辑碎片，他们将决定你构建的区块链可以做什么！ Substrate 在 FRAME 框架中内置了大量的预建模块。

例如， FRAME 中有一个Balances 模块，他可以通过管理您的系统中所有账户的 余额 的方式，来控制您区块链的基础货币。

如果你想要在你的区块链中添加 "智能合约" 功能，你只需要引入 Contracts 模块。

像 "链上治理" 的功能，也可以通过在你的区块链中引入Democracy，Elections，和 Collective 等模块来实现。

本教程的目标是教您如何创建自己的 Substrate 模块，并引入到自己的区块链中！ Substrate 开发者平台的 Node Template 附带一个模板 (template) 模块，您可将它视作为一个起点，在上面构建自定义的 Runtime 逻辑。

文件结构

在您喜欢的代码编辑器中打开 Node Template，然后打开文件 pallets/template/src/lib.rs

substrate-node-template
|
+-- node
|
+-- pallets
|   |
|   +-- template
|       |
|       +-- Cargo.toml    <-- One change in this file
|       |
|       +-- src
|           |
|           +-- lib.rs     <-- Most changes in this file
|           |
|           +-- mock.rs
|           |
|           +-- tests.rs
|
+-- runtime
|
+-- scripts
|
+-- ...

您将看到一些预置的代码作为新的模块模板。 如果您喜欢，可以通读这个文件。然后删除内容，因为接下来我们将从零开始完整、清楚的学习。 在将来撰写您自己的 pallets 时，您将发现这个模板钱包的脚手架是非常有用的。

构建您的新模块

宏观来看，Substrate 模块可以分为六个部份：

// 1. 导入
use frame_support::{decl_module, decl_storage, decl_event, decl_error, dispatch};
use frame_system::ensure_signed;

// 2. 配置
pub trait Trait: frame_system::Trait { /* --snip-- */ }

// 3. 存储
decl_storage! { /* --snip-- */ }

// 4. 事件
decl_event! { /* --snip-- */ }

// 5. 错误
decl_error! { /* --snip-- */ }

// 6. 可调用函数
decl_module！ { /* --snip-- */ }

如果您做过其他区块链的开发，那么事件、存储和可调用函数等概念会对您来说并不陌生。 我们将向逐一向您展示这些组件， 如基础的 "存在证明" 模块.

导入和依赖关系

由于导入的内容相当繁琐，您可以先将其粘贴在空的 lib.rs 文件的顶部：

#![cfg_attr(not(feature = "std"), no_std)]

use frame_support::{
    decl_module, decl_storage, decl_event, decl_error, ensure, StorageMap
};
use frame_system::ensure_signed;
use sp_std::vec::Vec;

这些导入的内容大多已经可用，因为它们被用于模板钱包，我们刚刚删除了它的代码 。 然而， sp_std 不可用，我们需要将其列为依赖项。

添加 这块代码到 pallets/template/Cargo.toml 文件中.

[dependencies]
#--snip--
sp-std = { default-features = false, version = '2.0.0' }

然后 更新 现有的 [features] ，就像这样。 最后一行是新的。 在下一个教程 添加一个模块 中，您将了解到为什么需要这么做。

[features]
default = ['std']
std = [
    'codec/std',
    'frame-support/std',
    'frame-system/std',
    'sp-std/std',          <-- This line is new
]

配置

每个模块都有一个称为 Trait 的组件，用于做相关配置。 此组件属于 Rust 語言的 "trait" 范畴；在 Rust 中，traits 类似于 C ++，Java 和 Go 等语言的接口 (Interface)。 现在，我们唯一需要配置的，是我们 Pallet 将要触发的一些事件。 我们将在下一个教程 添加一個模块 裡进行更深入的介绍 Trait 接口。

/// 通过指定它所依赖的参数和类型来配置模块。
pub trait Trait: frame_system::Trait {
    /// 因为这个模块触发事件，所以依赖于 Runtime 里事件的定义。
    type Event: From<Event<Self>> + Into<<Self as frame_system::Trait>::Event>;
}

事件

在我们配置了模块触发的事件后，让我们继续定义这些事件：

// 当做出重要更改时，模块通过事件通知用户。
// 事件文档应该以提供参数描述名称的数组结尾。
// https://substrate.dev/docs/en/knowledgebase/runtime/events
decl_event! {
    pub enum Event<T> where AccountId = <T as frame_system::Trait>::AccountId {
        /// Event emitted when a proof has been claimed. [who, claim]
        ClaimCreated(AccountId, Vec<u8>),
        /// Event emitted when a claim is revoked by the owner. [who, claim]
        ClaimRevoked(AccountId, Vec<u8>),
    }
}

我们的模块只会在这两种情况下发布事件：

  1. 当一个新的证明被添加到区块链中。
  2. 当证明被移除时。

事件可以包含一些元数据，在这种情况下，每个事件中也会显示是谁触发了事件 (AccountId)， 和已储存或移除的证明数据( Vec<u8>)。 请注意，在事件文档的末尾，按照惯例要包含一个对这些参数具描述性的名称。

错误

如果先前定义的事件被发出，那代表着我们已成功完成模块的函数调用。 同样的，当一个调用失败后将发出错误提示，并标明为何调用失败。

// 错误通知：出错时对用户进行通知
decl_error! {
    pub enum Error for Module<T: Trait> {
        /// 该证明已经被声明
        ProofAlreadyClaimed,
        /// 该证明不存在，因此它不能被撤销
        NoSuchProof,
        /// 该证明已经被另一个账号声明，因此它不能被撤销
        NotProofOwner,
    }
}

在试图声明新的存证时，可能发生上述的第一个错误。 当然，用户不可以声明一个已经被声明过的存证。 后两个错误，可能在试图撤销证明时发生。

Storage

为了向区块链添加新的存证，我们只需将此证明储存在我们模块的存储里。 要存储该值，我们需要创建一个 哈希映射 ，包含存证的拥有人和存证所在的区块号。

//  模块的 runtime 存储项目。
// https://substrate.dev/docs/en/knowledgebase/runtime/storage
decl_storage! {
    trait Store for Module<T: Trait> as TemplateModule {
        /// 证明的存储项目
        /// 它将证明映射到提出声明的用户以及声明的时间。
        Proofs: map hasher(blake2_128_concat) Vec<u8> => (T::AccountId, T::BlockNumber);
    }
}

如果存证有着一个拥有人和区块号，那么我们可判断出它已经被声明过了！ 否则，存证就可以被其他人声明。

可调用函数

正如我们模块中的事件和错误所提示的那样，我们将有两个“可调用函数”，用户可以在这个 FRAME 模块中调用：

  1. create_claim()：允许用户通过上传文件摘要来声明存在一个文件。
  2. revoke_claim()：允许存证的拥有人撤销他们的拥有权。
// 可调用函数允许用户与模块交互并调用状态更改。
// 这些函数将实现为“Extrinsics”，它经常被拿来和交易做对比。
// 可调用函数必须标注 Weight 并返回调用结果。
decl_module! {
    pub struct Module<T: Trait> for enum Call where origin: T::Origin {
        // 错误必须被初始化，如果它们被 Pallet 所使用。
        type Error = Error<T>;

        // 事件必须被初始化，如果它们被模块所使用。
        fn deposit_event() = default;

        /// 允许用户队未声明的证明拥有所有权
        #[weight = 10_000]
        fn create_claim(origin, proof: Vec<u8>) {
            // 检查 extrinsic 是否签名并获得签名者
            // 如果 extrinsic 未签名，此函数将返回一个错误。
            // https://substrate.dev/docs/en/knowledgebase/runtime/origin
            let sender = ensure_signed(origin)?;

            // 校验指定的证明是否被声明
            ensure!(!Proofs::<T>::contains_key(&proof), Error::<T>::ProofAlreadyClaimed);

            // 从 FRAME 系统模块中获取区块号.
            let current_block = <frame_system::Module<T>>::block_number();

            // 存储证明：发送人与区块号
            Proofs::<T>::insert(&proof, (&sender, current_block));

            // 声明创建后，发送事件
            Self::deposit_event(RawEvent::ClaimCreated(sender, proof));
        }

        /// 允许证明所有者撤回声明
        #[weight = 10_000]
        fn revoke_claim(origin, proof: Vec<u8>) {
            //  检查 extrinsic 是否签名并获得签名者
            // 如果 extrinsic 未签名，此函数将返回一个错误。
            // https://substrate.dev/docs/en/knowledgebase/runtime/origin
            let sender = ensure_signed(origin)?;

            // 校验指定的证明是否被声明
            ensure!(Proofs::<T>::contains_key(&proof), Error::<T>::NoSuchProof);

            // 获取声明的所有者
            let (owner, _) = Proofs::<T>::get(&proof);

            // 验证当前的调用者是证声明的所有者
            ensure!(sender == owner, Error::<T>::NotProofOwner);

            // 从存储中移除声明
            Proofs::<T>::remove(&proof);

            // 声明抹掉后，发送事件
            Self::deposit_event(RawEvent::ClaimRevoked(sender, proof));
        }
    }
}

你在这里看到的函数没有明确说明返回的类型。 事实上，它们都返回了 DispatchResult. 这个返回类型是 decl_module! 宏代你添加的。

编译您的新模块

在您将此模块的所有部分正确复制到您的 pallets/template/lib.rs 文件后，您应该能够重新编译您的节点，而且没有警告或报错。 在 substrate-node-template 根目录下运行此命令来构建和运行节点：

WASM_BUILD_TOOLCHAIN=nightly-2020-10-05 cargo run --release -- --dev --tmp

现在是与我们新的存证模块交互的时候了！
