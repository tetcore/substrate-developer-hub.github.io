安装

此页面将引导您准备好 Substrate 开发所需的环境。 由于 Substrate 是用 Rust 语言构建的，所以你需要做的第一件事是为 Rust 准备开发环境--这些步骤将根据计算机的操作系统而变化。 配置好 Rust 开发环境后，您将使用它的 toolchain 与 Rust 项目进行交互；所有支持或基于 Unix 的操作系统的 Rust 的 toolchain 命令都是相同的。

基于 Unix 的操作系统

在基于 Unix 的操作系统，例如 macOS 或 Linux 上，进行 Substrast 开发最为容易。 在 Substrate Tutorials 和 Recipes 中的例子，使用了 Unix 风格的终端来演示，如何使用命令行与 Substrate 交互。

macOS

打开 Terminal 终端应用程序并执行以下命令：

# Install Homebrew if necessary https://brew.sh/
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install.sh)"

# Make sure Homebrew is up-to-date, install openssl and cmake
brew update
brew install openssl cmake

Ubuntu / Debian

使用 terminal 终端应用程序执行以下命令：

sudo apt update
# May prompt for location information
sudo apt install -y cmake pkg-config libssl-dev git build-essential clang libclang-dev curl libz-dev

Arch Linux

在 terminal 中运行如下命令：

pacman -Syu --needed --noconfirm cmake gcc openssl-1.0 pkgconf git clang
export OPENSSL_LIB_DIR="/usr/lib/openssl-1.0"
export OPENSSL_INCLUDE_DIR="/usr/include/openssl-1.0"

Windows

请参阅 Windows 用户指南。

Rust 开发者环境

本指南使用 rustup 来帮助管理 Rust 的 toolchain。 首先安装并配置 rustup:

# Install
curl https://sh.rustup.rs -sSf | sh
# Configure
source ~/.cargo/env

配置 Rust 的 toolchain 为默认的最新稳定版本：

rustup default stable

WebAssembly 编译

Substrate 使用 WebAssembly(Wasm) 来生成可迁移的区块链 Runtime。 您需要配置 Rust 编译器来使用 nightly builds ，以允许为 Wasm 的目标编译 Substrate Runtime 代码。

Rust Nightly Toolchain

使用 Substrate 进行构建的开发者应该使用与其 Substrate 版本兼容的特定 Rust nightly 版本；这个版本会因项目而异，不同的项目可能会使用不同的机制向开发者告知其 Rust nightly 版本。 例如，Polkadot 客户端在其 发布说明 中告知了此信息。 Substrate 节点模板使用了一个 init script 和 Makefile 来指定 Rust nightly 版本并封装了后续步骤。 使用 Rusup 安装正确的 nightly 版本：

rustup install nightly-<yyyy-MM-dd>

Wasm Toolchain

配置 nightly 版本与 Wasm 的编译目标一起工作：

rustup target add wasm32-unknown-unknown --toolchain nightly-<yyyy-MM-dd>

指定 Nightly 版本

使用 WASM_BUILD_TOOLCHAIN 环境变量来指定 Substrate 项目在 Wasm 编译时应使用的 Rust nightly 版本：

WASM_BUILD_TOOLCHAIN=nightly-<yyyy-MM-dd> cargo build --release

请注意，这只是用指定的 nightly 构建_ Runtime_。 其余的项目将使用默认的 toolchain 进行编译，也就是最新的稳定 toolchain 版本。

Substrate 最新的 Nightly 版本 master

正在构建 Substrate itself 的开发者应该始终使用最新的 Rust stable 和 nightly 的无 bug 版本。 由于 Substrate 代码库依赖于 Rust Nightly 的一些小特性，即上游的 Rust nightly 编译器的部分修改通常将导致 Substrate 进行变更。 为了确保你的 Rust 编译器是最新，请运行：

rustup update
rustup update nightly
rustup target add wasm32-unknown-unknown --toolchain nightly

** 若上游 Substrate 代码库的变化依赖于 Rust 编译器的新特性，可能需要偶尔重新运行 rustup update**。

Rust Nightly 降级

若你的电脑已配置最新版的 Rust nightly，而你想将其降级至某个特定版本，请遵照如下步骤：

rustup uninstall nightly
rustup install nightly-<yyyy-MM-dd>
rustup target add wasm32-unknown-unknown --toolchain nightly-<yyyy-MM-dd>

测试

按照我们第一个教程 Create Your First Substrate Chain 中的步骤，是确保你成功部署好 Substrate 开发使用的计算机的最好方法。
