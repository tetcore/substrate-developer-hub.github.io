ink! 概念

ink! 是基于 Rust 的嵌入式域特定语言（eDSL），用于编写在 FRAME Contracts 模块 里运行的 Wasm 智能合约。

设计

Ink! 的主要目的 是正确性、简洁性、和高效性。

ink! 设计上尽可能接近 Rust 编程语言。 该语言使用 属性宏 将标准的 Rust 结构标记为可理解的合约组件。

#[ink(...)]

因为 ink! 遵循 Rust 标准，开发者可直接使用像 rustfmt 和 rust-analyzer 这类的工具。

合约组件

ink! 对使用其他现代智能合约语言编程的开发者来说应该还是很熟悉的。 合约的结构中具有开发者预期的所有组件：

  • 事件 (Events)
  • 事件主题 (Event Topics)
  • 存储 (Storage)
  • 构造函数 (Constructor Functions)
  • 消息函数 (Message Functions)

溢出保护

因为是采用 Rust 编写，ink! 可以提供编译时的溢出/下溢安全保护。 通过调整 rust 编译器的配置，你可以自定义是否可进行溢出计算，或在溢出时让合约发出 panic。 而无需频繁的导入 "Safe Math" 库，尽管 Rust 也提供了 综合检查，封装，及溢满的数学函数和功能。

注意：在编译器级的溢出检查以及编译后 Wasm 二进制代码的大小方面，存在一些已知的问题。 这些功能将来可能会改变或迭代。

测试环境

ink! 提供了一个内置的测试环境，可用于使用 Rust 框架执行链外单元测试。 这样就可以简单方便地确保你的合约代码按预期功能运行，而不需要引入第三方测试平台。

ink! 与 Solidity 的对比

Rust 是一种理想的智能合约语言。 它是类型安全，内存安全，并且没有定义模糊的语言行为。 它生成的二进制代码很小，因为它不包含额外的冗余功能，比如垃圾回收、高级优化和 tree-shaking 来移除无用代码。 通过编译器的标志，Rust 可以实现整数溢出自动保护。

ink! 选择改编 Rust 的一个子集来达到这个目的，而不是发明一种新的编程语言。 因此你自动获得了 Rust 生态系统的所有工具和支持。 此外，随着语言的发展，ink! 将自动获得新的特性和功能，从而提升将来开发者编写智能合约的能力。
